## What I Did

### Step 1 — iDRAC Access & Reset
> The server had unknown login credentials for both the OS and the iDRAC remote management interface. Had to gain access before anything else.

- Used an **IP scanner tool** to scan the network and identify the iDRAC IP address — identified by the iDRAC hostname label in the scan results
- Attempted to log into iDRAC using:
  - Default credentials (root / calvin)
  - Credentials listed on the physical credential card attached to the server
  - Neither worked
- Researched the issue and determined the best path forward was a **factory reset of iDRAC settings**
- Accessed the server's **BIOS** and reset iDRAC configuration to factory defaults
- Logged in successfully with default credentials after the reset
- Set a new secure password and **documented the credentials** in IT Glue for future reference

### Step 2 — Windows Server 2025 Installation
> With iDRAC sorted, moved on to wiping and reinstalling the OS.

- Used **Rufus** to flash a **Windows Server 2025 ISO** onto a USB drive, creating a bootable installer
- Entered the server **BIOS** and modified the boot sequence to prioritize the USB drive
- Booted from the USB and performed a clean installation of **Windows Server 2025** on the Dell PowerEdge R640
- Installation completed successfully

### Step 3 — DC Role Configuration via SOP
> After OS installation, followed the company's SOP guide to configure the server as a temporary Domain Controller for client migration.

- Worked through the SOP guide alongside my coworker who double-checked the configuration
- Installed the necessary **Active Directory Domain Services (AD DS)** roles
- Applied required configurations to prepare the server for the client's domain migration
- Server prepared and ready to be used as a temporary DC

## What I Learned
Today was the first time I've touched a physical enterprise server — a Dell PowerEdge R640 — and it was a completely different experience from working in a home lab VM. The iDRAC experience alone was a valuable lesson: enterprise servers have an entire out-of-band remote management layer (iDRAC) that operates independently of the OS, and knowing how to access, reset, and configure it is a real sysadmin skill. I also learned that when default credentials don't work and documented credentials fail, a factory reset is often the cleanest path forward — and documenting the new credentials immediately after is non-negotiable. Flashing a server OS from a Rufus bootable USB and modifying the boot sequence in BIOS are tasks I'd done in a home lab context before, but doing it on production hardware for a real client migration made it feel completely different.
