## What I Did

### Multi-Tenant MFA Audit — Microsoft Entra ID

> Tasked with auditing authentication methods across all client tenants managed by Captain IT through Microsoft Entra ID.

- Accessed each managed client tenant through the **Microsoft Entra ID** admin portal
- Navigated to authentication method settings for each tenant
- Retrieved and documented the **primary authentication methods** in use across all users in each tenant
- Specifically flagged users with **SMS (text message) set as their primary MFA method**
- SMS is being deprecated by Microsoft as a primary authentication method due to known security vulnerabilities including:
  - **SIM swapping attacks** — where an attacker hijacks a victim's phone number to intercept SMS codes
  - **SS7 protocol vulnerabilities** — weaknesses in the underlying telecom signaling system that can allow interception of SMS messages
- Data collected will be used to inform clients and plan migration away from SMS-based MFA to more secure methods such as Microsoft Authenticator or FIDO2 security keys

## What I Learned
This task gave me direct exposure to Identity and Access Management (IAM) at the tenant level — something that is a core discipline in cybersecurity. Seeing firsthand how many users across multiple organizations are still relying on SMS for MFA reinforced why this is an active security concern, not just a theoretical one. SMS-based MFA is better than no MFA, but it is vulnerable to SIM swapping and SS7 interception — and Microsoft phasing it out is the right move from a security standpoint. This is the kind of proactive security audit work that security analysts and cloud security engineers do regularly, and getting hands-on experience with it through Entra ID in a real MSP environment is directly relevant to my goal of moving into cloud security.
